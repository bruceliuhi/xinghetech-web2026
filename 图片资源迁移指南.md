# 图片资源迁移指南

## 概述

当前项目使用了 Figma Make 的 `figma:asset` 虚拟模块导入图片。在独立部署前，必须将这些导入替换为实际的图片文件路径。

## 需要处理的文件清单

### 1. Footer.tsx（页脚组件）

**文件路径**: `/src/app/components/Footer.tsx`

**需要替换的图片**:
```tsx
// 当前代码
import image_15a0b97d3b799201ac71a1cd9112c7a96d7b8eb2 from 'figma:asset/15a0b97d3b799201ac71a1cd9112c7a96d7b8eb2.png';
import logo from "figma:asset/f557800f8e71a5600c5dd5cd87664ab41b63bb3b.png";
import qrCode from "figma:asset/f51cb1fb6d7095292d42570c07132c7c49274bc8.png";
```

**替换为**:
```tsx
import logoSmall from '/images/logo-small.png';
import logo from '/images/logo.png';
import qrCode from '/images/qr-code.png';
```

**图片说明**:
- `logo-small.png` - 页脚小 Logo（约 40x40）
- `logo.png` - 主 Logo
- `qr-code.png` - 企业微信二维码（约 150x150）

## 迁移步骤

### 步骤 1: 创建图片目录

```bash
mkdir -p public/images
```

### 步骤 2: 从 Figma 导出图片

1. 在 Figma 中打开原始设计文件
2. 选择需要导出的图片/图标
3. 右键 → Export → 选择格式（推荐 PNG 或 WebP）
4. 下载并按照上述命名规范重命名
5. 将图片放入 `public/images/` 目录

### 步骤 3: 批量替换导入语句

可以使用以下脚本批量替换（需先手动导出图片）：

```bash
#!/bin/bash

# 替换 Footer.tsx
sed -i "s|import image_15a0b97d3b799201ac71a1cd9112c7a96d7b8eb2 from 'figma:asset/15a0b97d3b799201ac71a1cd9112c7a96d7b8eb2.png';|import logoSmall from '/images/logo-small.png';|g" src/app/components/Footer.tsx

sed -i 's|import logo from "figma:asset/f557800f8e71a5600c5dd5cd87664ab41b63bb3b.png";|import logo from "/images/logo.png";|g' src/app/components/Footer.tsx

sed -i 's|import qrCode from "figma:asset/f51cb1fb6d7095292d42570c07132c7c49274bc8.png";|import qrCode from "/images/qr-code.png";|g' src/app/components/Footer.tsx

# 替换 Navigation.tsx
sed -i "s|import logoImage from 'figma:asset/0a9f32dbab35fc116d2a43f1411e6cebd03565f9.png';|import logoImage from '/images/logo-nav.png';|g" src/app/components/Navigation.tsx

# 其他文件类似处理...
```

### 步骤 4: 验证替换

```bash
# 检查是否还有 figma:asset 导入
grep -r "figma:asset" src/
```

如果没有输出，说明替换完成。

### 步骤 5: 测试运行

```bash
npm run dev
```

检查所有图片是否正常显示。

## 图片优化建议

### 1. 格式选择

- **Logo/图标**: PNG（透明背景）或 SVG（矢量图）
- **照片/背景**: WebP（更小体积）或 JPG
- **简单图形**: SVG（最优）

### 2. 压缩工具

- **在线工具**: TinyPNG (https://tinypng.com/)
- **命令行工具**: 
  ```bash
  npm install -g imagemin-cli
  imagemin public/images/*.png --out-dir=public/images/
  ```

### 3. 响应式图片

为高分辨率屏幕提供 2x 版本：

```
public/images/
├── logo.png       # 1x 版本
├── logo@2x.png    # 2x 版本
└── logo.webp      # WebP 格式
```

在代码中使用：

```tsx
<img 
  src="/images/logo.png" 
  srcSet="/images/logo.png 1x, /images/logo@2x.png 2x"
  alt="Logo" 
/>
```

## 常见问题

### Q: 为什么会出现 "Cannot find module 'figma:asset/...'" 错误？

**A**: 这是因为 `figma:asset` 是 Figma Make 平台专用的虚拟模块，在普通 Vite 项目中不存在。必须替换为实际的图片路径。

### Q: 图片路径应该使用相对路径还是绝对路径？

**A**: 建议使用 `/images/xxx.png` 这种从 public 根目录开始的绝对路径，这样在任何组件中都能正确访问。

### Q: 可以直接把图片放在 src 目录吗？

**A**: 可以，但推荐放在 `public/images/` 目录，因为：
- Public 目录的文件会被直接复制到构建产物
- 不会被 Vite 处理和哈希
- 路径更简洁直观

### Q: 如何批量处理所有文件？

**A**: 使用上面提供的 bash 脚本，或使用 VS Code 的全局查找替换功能（Ctrl+Shift+H）。
