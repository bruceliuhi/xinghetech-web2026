## ⚠️ 重要：图片资源必须先处理！

### 为什么必须处理图片？

当前项目使用了 Figma Make 专用的 `figma:asset` 虚拟模块导入图片。这在独立部署时**不会工作**，必须替换为实际图片路径。

### 需要处理的图片（13 个）

```
public/images/
├── logo.png                    ← Logo 相关（3个）
├── logo-small.png
├── logo-nav.png
├── qr-code.png                 ← 二维码（1个）
├── bg-agent.png                ← 背景图（3个）
├── bg-background.png
├── bg-practices.png
├── icon-chat-3d.png            ← 3D图标（6个）
├── icon-cluster-3d.png
├── icon-report-3d.png
├── icon-knowledge-3d.png
├── icon-doc-3d.png
├── icon-flow-3d.png
└── decorative-scenarios.png    ← 装饰图（1个）
```

### 处理步骤

1. **阅读**: `图片资源迁移指南.md`
2. **创建目录**: `mkdir -p public/images`
3. **从 Figma 导出图片** 或 **使用占位图**
4. **替换代码中的导入语句**
5. **验证**: `grep -r "figma:asset" src/` 应无结果

**详细步骤**: 见 `图片资源迁移指南.md`

## 🚀 推荐部署流程（首次部署）

### 第一步：准备环境（本地）

```bash
# 1. 克隆项目
git clone <your-repo-url>
cd xinghe-ai-website

# 2. 安装依赖
npm install

# 3. 创建图片目录
mkdir -p public/images

# 4. 从 Figma 导出图片到 public/images/
# （或使用占位图快速测试）

# 5. 替换代码中的 figma:asset 导入
# 参考：图片资源迁移指南.md

# 6. 本地测试
npm run dev
# 访问 http://localhost:5173 检查所有图片

# 7. 本地构建测试
npm run build
npm run preview
```

### 第二步：部署到服务器

**选择一种方式**:

#### A. 自动脚本部署（推荐）

```bash
# 1. 上传代码到服务器
scp -r . user@server:/var/www/xinghe-ai

# 2. SSH 登录服务器
ssh user@server

# 3. 执行部署
cd /var/www/xinghe-ai
sudo ./deploy.sh
```

#### B. Docker 部署

```bash
# 1. 上传代码到服务器
scp -r . user@server:/var/www/xinghe-ai

# 2. SSH 登录服务器
ssh user@server

# 3. 构建并运行
cd /var/www/xinghe-ai
docker-compose up -d
```

### 第三步：配置域名和 HTTPS

```bash
# 1. 配置域名解析（在域名服务商）
# A 记录: your-domain.com → 服务器IP

# 2. 配置 Nginx
sudo nano /etc/nginx/sites-available/xinghe-ai
# 修改 server_name 为你的域名

# 3. 安装 SSL 证书
sudo certbot --nginx -d your-domain.com

# 4. 测试
curl -I https://your-domain.com
```

### 第四步：验证部署

参考 `部署检查清单.md` 逐项检查：

- [ ] 所有页面正常访问
- [ ] 图片全部加载
- [ ] 路由正常工作
- [ ] HTTPS 正常
- [ ] 性能达标

## 🔧 常用命令

### 开发命令

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build

# 预览构建产物
npm run preview

# 检查图片导入
grep -r "figma:asset" src/
```

### 部署命令

```bash
# 自动部署脚本
sudo ./deploy.sh

# Docker 部署
docker-compose up -d
docker-compose down
docker logs xinghe-ai-web

# Nginx 操作
sudo nginx -t                    # 测试配置
sudo systemctl reload nginx      # 重载配置
sudo systemctl restart nginx     # 重启服务
```

### 监控命令

```bash
# 查看 Nginx 日志
tail -f /var/log/nginx/access.log
tail -f /var/log/nginx/error.log

# 查看服务器状态
top
df -h
free -h
```
